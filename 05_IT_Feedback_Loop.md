# 05 IT 反馈与维护 (IT Feedback Loop)

> 建议收集表单模板、系统维护期间的替代方案。阅读时间约 8 分钟。

---

## 1. 建议收集表单模板 (Feedback Collection / フィードバック収集)

用于：客服/一线将客户或自身遇到的系统问题、改进建议，标准化提交给 IT 或产品团队。

### 1.1 表单字段（可做成工单/在线表单项）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| **提交人** | 文本 | 是 | 客服工号或部门 |
| **提交日期** | 日期 | 是 | 建议提交当天 |
| **类别** | 单选 | 是 | 故障 (Bug) / 体验优化 (UX) / 功能建议 (Feature) / 其他 |
| **涉及模块** | 单选/多选 | 是 | 登录 / 入金出金 / 交易下单 / 账户设置 / 报表 / 其他 |
| **问题或建议描述** | 长文本 | 是 | 客观描述现象或建议，避免主观情绪化表述 |
| **复现步骤** | 长文本 | 建议 | 若为故障，请按步骤列出操作路径 |
| **客户反馈摘要** | 长文本 | 否 | 客户原话或概括（脱敏，不包含个人身份信息） |
| **发生频率/影响范围** | 文本 | 否 | 如：偶发/必现、单客户/多客户 |
| **附件** | 文件 | 否 | 截图、日志（脱敏）等 |

### 1.2 填写示例

```
类别：故障 (Bug)
涉及模块：交易下单
问题描述：客户在 App 端市价单提交后，订单状态长时间显示「处理中」，约 2 分钟后才显示「已成交」或「已拒绝」，客户担心重复下单。
复现步骤：1) 登录 App  2) 选择品种 xxx  3) 选择市价单、输入手数  4) 提交  5) 观察订单状态
发生频率：客户反馈当日多次出现，非必现。
客户反馈摘要：「点了好几次都不知道有没有成交，不敢再点。」
```

### 1.3 话术建议（向客户收集信息时）

| 类型 | 推荐用语 | 禁忌用语 |
|------|----------|----------|
| 邀请反馈 | 「您遇到的问题我们会记录并转交技术团队优化。若方便，请简单描述一下操作步骤或发一张截图，便于我们更快定位。」 | 「我们只负责记录，改不改不知道」 |
| 确认跟进 | 「您的建议已提交，工单号 xxx。技术团队会评估后纳入排期，有进展我们会通过内部流程跟进。」 | 「保证会改」「马上修」 |

## 2. 系统维护期间的替代方案 (Alternatives During Maintenance / メンテナンス時の代替案)

当计划内维护导致部分功能不可用时，客服可提供的替代路径与话术。

### 2.1 维护前（公告与提醒）

| 动作 | 说明 |
|------|------|
| 公告渠道 | 官网、App 推送、邮件（针对活跃用户）、客服端公告 |
| 公告要点 | 维护时间（起止）、影响范围（如：无法登录、无法出金）、预计恢复时间、紧急联系渠道（如有） |
| 客服口径 | 「平台将于 [时间] 进行系统维护，期间 [影响说明]。请您提前安排交易与出金，具体以官方公告为准。」 |

### 2.2 维护期间可提供的替代方案

| 受影响功能 | 替代方案（示例） | 话术要点 |
|------------|------------------|----------|
| **无法登录/交易** | 无直接替代；提醒客户在维护结束后再操作，避免重复尝试 | 「当前为计划内维护，预计 [结束时间] 恢复。请您在恢复后再登录或交易，避免在异常状态下操作。」 |
| **无法出金** | 告知维护结束后按申请顺序处理；若有加急通道，按内部规定执行 | 「出金功能在维护期间暂停，恢复后将按提交顺序处理。若您有紧急需求，可留下联系方式，我们会在恢复后优先跟进。」 |
| **无法查看持仓/报表** | 若后台可查，仅限内部使用，不承诺向客户代查；可告知「恢复后可自行查看」 | 「报表与持仓在维护结束后会恢复正常显示，请您稍后重试。我们无法代您操作账户或代查实时数据。」 |
| **客服系统本身维护** | 提供备用联系方式（如仅邮件、或指定时段电话），并在公告中说明 | 「客服系统正在维护，您可通过 [备用邮箱/链接] 留言，我们会在恢复后尽快回复。」 |

> [!CAUTION]
> 维护期间不得承诺「一定在 X 点恢复」或「保证不影响您的订单」，仅以官方公告的预计时间为准，避免因延期引发客诉。

### 2.3 话术建议

| 类型 | 推荐用语 | 禁忌用语 |
|------|----------|----------|
| 说明原因 | 「当前为计划内系统维护，部分功能暂时不可用，预计 [时间] 恢复，请您以公告为准。」 | 「系统坏了」「不知道什么时候好」 |
| 替代指引 | 「维护期间无法 [具体功能]，您可在恢复后 [操作]。如有紧急情况可留下联系方式，我们恢复后优先联系您。」 | 「没办法，只能等」 |
