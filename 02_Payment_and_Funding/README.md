# 02 入金与出金 (Payment and Funding)

> USDT/Peska 流程、支付失败转化策略、未到账对账标准、出金标准场景的统一作业规范。涉及合规（AML）与风控，回复须**明确规则、原因及下一步操作**。

---

## 核心要点（基础客服必读）

| 主题 | 核心要点 | 严禁 |
|------|----------|------|
| **1. USDT 入金** | 链类型确认（如 USDT 用 TRC20）；地址复制粘贴勿手输；「支付金额」栏**仅限数字**。到账约 10 分钟内，可在入金记录查看 | 链选错、手输地址、金额栏输入币种/链名；未核实即承诺「已到账」 |
| **2. Peska 流程** | AFT 与 Peska 的**姓名（罗马字）、邮箱、电话必须完全一致**。邮箱：我的资料→设置→更新电子邮箱；姓名：我的资料→注册信息→右上角编辑。修改后自动生效 | 信息不一致即承诺到账；代为修改客户信息；不引导客户自助修正 |
| **3. 支付失败转化** | 顺序：安抚情绪→解决痛点（绿色通道/USDT/专属指导）→留存钩子（补偿咨询 Leader、优先注销承诺）。Peska 不显示多为注册信息不符，引导修正后重试 | 不说明原因即拒绝；不提供替代方案；擅自承诺具体补偿金额 |
| **4. 未到账对账** | 核实前不承诺到账；法币 1–3 工作日、加密货币视网络。索要凭证：虚拟货币用 TXID，法币用带姓名/金额/日期/流水号的截图。无法核对时转财务/支付渠道 | 未核实即告知「已丢失」「无法找回」；接受无法核验的凭证 |
| **5. 出金标准场景** | **原路返回**：出金方式须与入金方式一致（AML）。有持仓时须说明可用保证金不足、强平风险，指引平仓或减额后重申请。审核 1–3 工作日；汇出后法币 3–5 工作日。已打款时说明通道延迟，超时可提供凭单 | 不写明客户入金方式即拒绝；拒绝时不说明具体原因；否认已打款、推责客户 |
| **6. Zendesk 文档** | 转发帮助中心链接时须按**客户语言**选用对应版本（中文 zh-cn / 日文 ja-jp / 英文 en-au） | 对日文/中文客户发英文链接等语言不一致 |
