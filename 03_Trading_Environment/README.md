# 03 交易环境与紧急波动 (Trading Environment)

> 行情异常、平台延迟、强平争议、临时维护及交易服务器故障的统一口径与模板化回复，确保对外信息一致、合规。

---

## 市场与流动性参考（内部口径 / 客服须知）

**对外说明报价与波动来源时，可统一引用以下表述，不承诺单一交易所或单一 LP。**

| 类别 | 说明（可引用） |
|------|----------------|
| **交易所 (Exchanges)** | 平台报价参考主流交易所行情，包括但不限于：**CME**（芝加哥商业交易所）、**纽交所 NYSE**（纽约证券交易所）、**伦敦市场**（伦敦证券交易所 LSE 等）等。 |
| **流动性提供方 (LP / Liquidity Providers)** | 流动性来自多家机构流动性提供方（如 Citadel、XTX、Finato、Jump Trading 等）的整合报价，平台不对单笔订单的报价来源做承诺。 |
| **口径原则** | 对客户：可称「主流交易所及多家流动性提供方」「多边市场报价」；**严禁**对外承诺「您的单子一定在 CME 成交」或指名某一家 LP。 |

---

## 核心要点（基础客服必读）

| 主题 | 核心要点 | 严禁 |
|------|----------|------|
| **1. 行情剧烈波动** | 说明为市场因素；可引用 **CME、纽交所、伦敦**等主流交易所及多家 **LP**（如 Citadel、XTX、Finato、Jump Trading 等）整合报价；不预判涨跌 | 预测涨跌、保证收益、「必涨/必跌」、承诺单笔订单在某某交易所/LP 成交 |
| **2. 平台延迟/卡顿** | 确认现象、安抚、收集账号/时间/截图并升级；未获技术确认前不否认系统问题 | 「系统从没出过问题」「是你网络不好」 |
| **3. 强平争议** | 按规则与记录复核；转风控/合规书面回复，不争论 | 「行情这样肯定要平」「我们不会错」 |
| **4. 临时维护** | 事前公告时间与范围；事中说明预计恢复时间，建议避免大额操作 | 未公告即停服、承诺具体恢复时间 |
| **5. 交易服务器故障** | 以官网/公告为准；安抚、记录影响范围与时间，转技术跟进；不承诺赔偿或到账时间 | 擅自承诺赔偿、对外宣称「没问题」 |
| **6. 点差扩大与滑点** | 归因于市场流动性/波动；浮动点差、滑点为全球普遍现象；婉拒赔偿时说明按市场价成交、无法干预 | 称「系统故障」；承诺补偿或撤单 |

## 使用原则

- **统一口径**：涉及行情异常、系统延迟、断线、强平等，均以本节模板或经合规/公关确认的表述为准。
- **不承诺行情与收益**：不得预测涨跌、保证收益或暗示「必涨」「必跌」。
- **留痕**：重要沟通建议保留记录，便于事后复盘与合规检查。
