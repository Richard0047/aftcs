# 5. 交易服务器故障 (Trading Server Failure / 取引サーバー障害)

> 交易服务器或 MT5 连接异常、无法下单/平仓时的统一口径、安抚与升级路径。

---

## 5.1 操作步骤

| 步骤 | 动作 | 说明 |
|------|------|------|
| 1 | 确认以公告为准 | 是否发生故障、影响范围与恢复时间以官网/App/内部公告为准，不擅自对外定性 |
| 2 | 安抚与记录 | 对客户致歉、说明正在核实/处理，记录客户账号、发生时间、现象（无法登录/无法下单/断线等） |
| 3 | 升级 | 转技术/运维跟进；对外口径需与合规/公关一致，不承诺具体恢复时间或赔偿 |

---

## 5.2 安全原则

| 要点 | 说明 |
|------|------|
| **不以个人名义定性** | 不得在未获技术/公告确认前对外宣称「服务器没问题」或「只是您本地网络问题」 |
| **不擅自承诺赔偿** | 是否补偿、补偿形式由公司政策与合规决定，客服不口头承诺「一定赔」或具体金额 |
| **留痕** | 记录客户反馈的时间、账号、设备与现象，便于技术排查与事后复盘 |

> [!CAUTION]
> 遇重大故障时，与合规/公关确认是否启用补充口径及对外公告内容。

---

## 5.3 话术建议

| 类型 | 推荐用语 | 禁忌用语 |
|------|----------|----------|
| 确认与安抚 | 「我们已收到您反馈的连接/交易异常，正在与技术部门核实。请您保留发生时间与操作截图，有进展会通过官网/邮件通知。」 | 「服务器没问题」「是你网络/设备问题」 |
| 升级 | 「您的情况已记录并转交技术团队。若确认为系统侧异常，我们会在官网/App 发布公告。恢复时间以公告为准。」 | 「马上修好」「X 点一定恢复」 |
| 赔偿/损失 | 「是否涉及补偿需由相关部门按公司政策与合规要求评估，有结论我们会正式通知您。」 | 「一定赔」「我们赔您 X」 |

## 5.5 场景：主动提供赔偿（经确认为我司/系统原因时）

**适用**：经相关部门确认，故障因与外部流动性供应商（LP）连接不稳定等**我司原因**导致，且存在客户无法平仓等实际损失时，使用以下口径主动说明原因并引导赔偿申请。**使用前须经合规/公关确认。**

### A. 故障说明与赔偿申请指引

#### 📋 复制即用 · 中文

```

针对本次情况，如果您因无法平仓而产生了实际损失，麻烦请您联系以下邮箱提供详细信息：

【关于赔偿的咨询窗口】
邮箱地址：global@afttmarkets.com
内容：账户号码、对应的订单编号、发生的情况描述

相关部门将进行严正调查，一旦确认是因我司原因导致的亏损，我们将竭诚为您办理赔偿手续。为了让您能够安心使用我们的平台，我们将全力以赴做好后续处理。
请多多关照。
```


### 使用注意

| 项目 | 说明 |
|------|------|
| **启用前提** | 仅当经内部确认故障为我司/系统原因（如与 LP 连接不稳定）且已获合规/公关同意对外使用赔偿口径时使用 |
| **赔偿窗口** | 赔偿咨询邮箱以公司最新公布为准（当前示例：global@afttmarkets.com）；客服不口头承诺具体赔偿金额或时间 |
| **留痕** | 发送前确认表述与公告一致；建议保留发送记录便于后续跟进 |
